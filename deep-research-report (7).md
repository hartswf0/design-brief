## I. PRE-PRODUCTION FRICTION  
```
From: Alex McDowell (Production Designer) <alex@futureworks>  
To: Henry LaBounta (ILM VFX Supervisor) <henry@ilm.com>  
Date: Nov 15, 2001  
Subject: PreCrime UI Prototype Specs  

Alex: “We must finalize the PreCrime console design. John needs the *‘sensitive glove’* input to gather PreCog vision data【6†L32-L39】.  As per Spielberg’s mandate: this is *‘future reality, not science fiction’*【22†L219-L223】, so the acrylic half-dome display has to be built for on-camera use.  The script calls for those engraved name-orbs (red/brown spheres) to appear with each vision【6†L32-L39】 – we need the props ready.  We’d need ~$80K more to prototype the wraparound screen and custom gloves.”  

Henry: “Budget is tight. We have **$150K** remaining for all set tech. A full 12×7 ft. curved plexi display is roughly $100K (safety glass).  ILM’s plan is to shoot Cruise against green walls and add all UI CG in post – on set we’ll use a 3×2' plexi mockup only.  (Also: Tom complained the gloves weren’t tracking smoothly during rehearsal – suggest checking marker placement.)”  
```

## III. TECHNICAL ADVISORY LOGS  
```
[Technical Advisory – CONFIDENTIAL, March 2002]

- **NeuroSync Glove Interface**: Sci.Advisor flagged the *precrime data grab* concept as scientifically baseless. (“Precogs are the innocents we now use to stop the guilty” – Hineman’s line – underscores ethical cost【32†L108-L116】). No known tech can ‘read’ dreams from an eye-viewer or project names onto spheres as shown.  
- **Temporal Paradox**: Captain Anderton sees his future murder – raises “no free will” contradiction. (As one memo paraphrases, *“if you show someone their future, they have no future”*【32†L97-L100】.) Script never addresses this causality issue.  
- **Device Specs [REDACTED]**: The *“Lidborg EyeScan”* device named in draft lacks any engineering basis – marked as (blacked out) in report.  
- **[CLASSIFIED – REDACTED]**: *Extensive sections of notes have been redacted under NDA (see *Appendix B*).*  
```  

## IV. POST-PRODUCTION REALITIES  
```
[dev.forum.thread.2017]

**User: DataArchitect** (09/15/2017 14:03): “Rewatching Minority Report’s PreCrime UI – just wondering what the underlying data model is.  Are those red/blue orbs actually database keys?  The movie never explains how a dream yields a suspect’s name.  Any insight from developers?”  

**Reply: GloveTech** (09/15/2017 15:20): “There really isn’t a coherent spec – it’s mostly movie flourish.  In fact, the production literally “built the world first” before finalizing story【22†L207-L215】.  One design retrospective even calls MR’s gesture UI *‘the de-facto way’* future interfaces were envisioned【39†L86-L94】.  So think of it as a visual metaphor: each gesture is like a query filter on a big crime database.  The orbs could represent DB records (suspects) that match the query.  In practice it’s predictive-policing + gestural UI, not a real technical system.”  
```
